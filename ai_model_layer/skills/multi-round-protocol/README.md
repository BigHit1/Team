# Multi-Round Protocol Skill

AI 多轮对话任务执行的标准化协议。

## 📁 文件说明

- **QUICKREF.md** - 快速参考卡片（1页）
- **SKILL.md** - 完整的协议文档
- **examples/** - 实际场景示例
- **templates/** - 可复用的状态模板

## 🎯 核心功能

通过在每次响应末尾输出结构化状态信息，实现：
- ✅ 自动追踪任务进度
- ✅ 明确何时需要人工介入
- ✅ 标准化错误处理
- ✅ 支持任务分步执行
- ✅ 提供可验证的完成状态

## 🚀 快速开始

### 基本格式

```yaml
---TASK_STATUS---
status: completed
reason: 任务完成
confidence: 0.9
---END_STATUS---
```

### 6种状态

1. **completed** - 任务完成（信心度 0.8-1.0）
2. **continue** - 需要继续（任务太大）
3. **need_human** - 需要人工介入（决策/许可/澄清）
4. **error** - 遇到错误（需要修复）
5. **waiting** - 等待中（等待外部操作）
6. **partial** - 部分完成（有警告，信心度 0.5-0.8）

## 📋 使用场景

- ✅ 复杂的代码生成任务
- ✅ 多文件修改操作
- ✅ 需要编译验证的任务
- ✅ 需要用户决策的任务
- ✅ 长时间运行的任务

## 🔑 关键规则

1. **始终输出状态块** - 每次响应末尾必须包含
2. **诚实评估信心度** - 不确定时不要虚报
3. **主动询问** - 不确定时使用 `need_human`
4. **提供详细信息** - 包含文件列表、错误信息等

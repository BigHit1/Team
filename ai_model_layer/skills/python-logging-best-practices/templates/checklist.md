# 日志质量检查清单

使用此清单检查代码中的日志质量。

## 📋 日志级别检查

### DEBUG
- [ ] 用于详细的调试信息
- [ ] 函数入口/出口
- [ ] 参数值和中间状态
- [ ] 不在生产环境输出到控制台

### INFO
- [ ] 用于重要的业务操作
- [ ] 任务开始/完成
- [ ] 关键步骤
- [ ] 配置信息

### WARNING
- [ ] 用于可恢复的问题
- [ ] 配置缺失（使用默认值）
- [ ] 性能警告（超过阈值）
- [ ] 弃用功能使用

### ERROR
- [ ] 用于操作失败
- [ ] 异常和错误
- [ ] 包含完整堆栈（`exc_info=True`）
- [ ] 包含上下文信息

### CRITICAL
- [ ] 用于系统级错误
- [ ] 系统无法启动
- [ ] 关键服务不可用

---

## 📊 日志内容检查

### 结构化数据
- [ ] 使用 `extra` 参数添加结构化数据
- [ ] 避免字符串拼接
- [ ] 数据类型正确（数字用数字，不用字符串）
- [ ] 键名使用下划线命名（snake_case）

### 上下文信息
- [ ] 包含关键标识符（task_id, user_id 等）
- [ ] 长任务使用上下文日志器
- [ ] 上下文信息完整且有意义

### 消息质量
- [ ] 消息清晰简洁
- [ ] 描述操作和结果
- [ ] 避免过多装饰性输出
- [ ] 避免重复信息

---

## ⚡ 性能监控检查

### 耗时记录
- [ ] 记录关键操作耗时
- [ ] 使用 `duration_seconds` 字段
- [ ] 超过阈值时记录 WARNING
- [ ] 正常情况记录 DEBUG

### 性能指标
- [ ] 记录处理数量
- [ ] 记录成功率
- [ ] 记录平均耗时
- [ ] 记录吞吐量

---

## 🐛 异常处理检查

### 异常记录
- [ ] 使用 `exc_info=True` 记录完整堆栈
- [ ] 记录错误类型（`type(e).__name__`）
- [ ] 记录错误消息（`str(e)`）
- [ ] 包含上下文信息

### 异常分类
- [ ] 预期错误使用 WARNING
- [ ] 非预期错误使用 ERROR
- [ ] 系统错误使用 CRITICAL
- [ ] 根据情况决定是否重新抛出

---

## 🔒 安全检查

### 敏感信息
- [ ] 密码已脱敏或不记录
- [ ] API Key 只记录前几位
- [ ] 邮箱已脱敏
- [ ] 个人信息已脱敏
- [ ] 不记录完整的信用卡号

### 数据量
- [ ] 大数据只记录摘要
- [ ] 长字符串截断（如 SQL 查询）
- [ ] 列表只记录长度或前几项
- [ ] 避免记录二进制数据

---

## 🎯 代码质量检查

### 日志器使用
- [ ] 使用 `get_logger(__name__)`
- [ ] 不使用 root logger
- [ ] 长任务使用上下文日志器
- [ ] 日志器命名正确

### 代码风格
- [ ] 避免在循环中过度日志
- [ ] 批量操作使用进度日志
- [ ] 避免重复日志
- [ ] 信息集中记录

### 初始化
- [ ] 应用入口调用 `setup_logging()`
- [ ] 配置合理的日志级别
- [ ] 配置日志轮转
- [ ] 配置日志目录

---

## 📈 可维护性检查

### 日志分析
- [ ] 日志便于查询
- [ ] 日志便于过滤
- [ ] 日志便于统计
- [ ] 支持 JSON 格式输出

### 调试友好
- [ ] 包含足够的上下文
- [ ] 错误信息清晰
- [ ] 便于问题定位
- [ ] 便于性能分析

---

## ✅ 检查结果

### 评分标准

- **优秀（90-100分）**: 所有检查项通过
- **良好（80-89分）**: 大部分检查项通过，少量需要改进
- **及格（70-79分）**: 基本检查项通过，部分需要改进
- **不及格（<70分）**: 多个检查项未通过，需要重构

### 改进优先级

**P0 - 必须修复**:
- 错误的日志级别
- 缺少异常堆栈
- 敏感信息泄露
- 关键操作缺少日志

**P1 - 应该修复**:
- 缺少上下文信息
- 日志消息不清晰
- 缺少性能监控
- 字符串拼接

**P2 - 可以优化**:
- 减少冗余日志
- 优化日志格式
- 添加更多调试信息
- 改进日志分析能力
